Parameter guide

Seed: Sets the pseudo-random branching stream. Same seed + settings gives the same trees. Different seeds change realized trees, so fitted exponents can shift.

Max depth: Maximum branching generations. Higher values allow longer paths and more terminal tips before growth is capped.

Main-branch bias (control): Preferred share of continuation along the main branch lineage during a split.

Asymmetry: How strongly that preference is enforced. At 0, splits are near symmetric; at 1, main-vs-side allocation is strongly skewed.

Smith path fraction (reported): $P_f = \bar{L_l}/L_l^*$, where $L_l^*$ is max root-to-tip path length. In ideal WBE-symmetric trees this approaches 1.0.

How to read these outputs

Fit: The red fitted slope from the plotted points for the currently simulated finite trees. It is an empirical regression result, not a theory value.

Asymptote: The estimated large-tree limit exponent b∞ from finite-size convergence. The simulator fits b_eff(S) = b∞ + c/S across tree-size classes and reports b∞ in this line.

Theory: A closed-form baseline exponent computed from the current geometry controls (length decay η and radius decay γ). It is WBE-style theory using your selected parameters, not a fixed "optimal WBE" constant unless your sliders are at WBE-optimal settings.

Finite-size: The fitted convergence model for observed effective slopes across size classes. S is tree size (tips, using class target tips), and c is the leading finite-size correction strength.

Is finite-size fit shown in scatter points? Not as a separate curve on the main scatter panel. Points are tree metrics; the finite-size relation is fit on slope-vs-size summaries and reported in the equation card.

Δfit: (finite-tree fit slope) − (theory slope).

Δasym: (asymptotic slope b∞) − (theory slope).
