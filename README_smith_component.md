# Smith Module README

This document describes the Smith et al. component added to this project, including:
- the R package module in `scalingPlantArchitecture`
- the separate browser visualization in `js-tree-simulator/smith-model.html`

It is intentionally separate from the existing SPA simulator and does not replace any prior workflow.

## Scope

The Smith component implements a rank-based tree model where:
1. topology is generated by recursive rank partitioning
2. diameters are assigned from rank
3. segment lengths are assigned from elastic-similarity path limits
4. architecture metrics and allometries are computed from generated trees

Core equations used in code:

- Rank conservation:
  - `R_m = sum_i R_{d,i}`
- Diameter from rank:
  - `D(R) = D_t * R^(1/2)`
- Distal supported path length:
  - `L*(D) = (b/s) * D^(2/3) - l0 * (b/s) * D_t^(2/3)`
- Internal segment length:
  - `l_m = L*(D_m) - L*(D_largest_daughter)`

### R quick start

```r
library(scalingPlantArchitecture)

p <- new_smith_params(
  N_t = 512,
  D_t = 0.02,
  f_max = 3,
  u = 0,
  safety_factor = 4,
  b_const = 10,
  l0_factor = 0.794,
  seed = 42
)

tr <- simulate_smith_tree(p)
compute_smith_path_metrics(tr)

# Single tree with Smith-style plotting
lay <- layout_smith_tree(tr, seed = 1042)
plot_smith_tree(tr, layout = lay, label = bquote(u == 0))

# Spectrum figure (fishbone to symmetric)
plot_smith_architecture_spectrum(
  N_twig = 512,
  f_max = 3,
  u_values = c(-5, -2, 0, 2, 5),
  seed = 42
)

# Save spectrum figure
pdf("smith_style_architecture_spectrum.pdf", width = 11, height = 4.5)
plot_smith_architecture_spectrum(
  N_twig = 512,
  f_max = 3,
  u_values = c(-5, -2, 0, 2, 5),
  seed = 42
)
dev.off()

# Allometry experiment
exp_out <- run_smith_allometry_experiment(
  n_tips_values = c(16, 32, 64, 128, 256),
  n_rep = 10,
  base_params = p
)
exp_out$fits
```

## Browser visualization

Location:
- `scaling plant architecture/js-tree-simulator/smith-model.html`
- `scaling plant architecture/js-tree-simulator/smith-model.js`
- `scaling plant architecture/js-tree-simulator/smith-model.css`

How to use:
1. Open `smith-model.html` in a browser.
2. Adjust controls for `N_t`, `D_t`, `f*`, `u`, `s`, `b`, and `virtual-length factor`.
3. Click **Generate Tree** for a single architecture.
4. Use **Run Allometry Sweep** to populate:
   - total path length vs tip count
   - total path length vs basal diameter

## Visual-style notes and what was improved

The Smith renderer now emphasizes the style seen in Smith et al. figures:
- tapered black stems for coarse woody structure
- gray fine branches for distal twigs
- lower `u` values yield narrower fishbone-like forms
- higher `u` values yield broader, more symmetric crowns

The style mapping is handled through angle spread and crown spread functions of `u` in both R and JS plotting layers.

## Current assumptions and limitations

- Furcation selection currently uses random choice in `[2, min(f_max, R_m)]` unless `furcation_mode = "max"`.
- Hydraulic conductance internals are not yet implemented as full conduit-level network physics in this module.
- The browser app prioritizes architecture and allometry diagnostics over full publication panel replication.
